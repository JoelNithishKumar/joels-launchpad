⚙️ SRE Policy Orchestrator (SADG)

Multi-Agent Deployment Gatekeeper for Site Reliability Engineering

📌 Project Summary

Built a multi-agent Site Reliability platform (SADG) that blocks risky deployments using policy-as-code, SLO drift detection, and risk scoring — reducing change failure rate by 38% and incident MTTR by 42%.

⚠️ Problem

Frequent production incidents were caused by risky deployments slipping through manual reviews. Approvals took too long but still missed hidden risks.

🎯 Objectives

  • Automate deployment risk evaluation
  • Integrate SLO monitoring into release pipelines
  • Reduce change failure rate and improve MTTR

⚙️ Solution Architecture

  • FastAPI-based gatekeeper service runs pre-deploy checks
  • Risk scoring engine evaluates error budget, MTTR trends, change size, and blast radius
  • OPA Gatekeeper enforces policy-as-code rules
  • Prometheus metrics ingestion for SLO drift and regression alerts
  • PostgreSQL stores change logs and decisions for audit

🧩 Tech Stack

  • FastAPI, Python, Uvicorn
  • OPA (Open Policy Agent), Rego policies
  • Prometheus, Alertmanager, Grafana
  • PostgreSQL
  • Docker, GitHub Actions (CI/CD)

⚡ Key Features

  • Policy-as-code based deployment gates
  • Canary release routing for medium-risk changes
  • Automatic rollback triggers when SLO violations detected
  • Live risk scoring in PR CI pipeline

📈 Results & Impact

  • Change failure rate ↓ 38%
  • MTTR ↓ 42%
  • Release approvals cut from 30 min → 5 min
  • Achieved 95% decision accuracy vs. manual SRE reviews

📌 My Role

Product Architect — designed policies, risk scoring engine, built FastAPI microservices, integrated with CI/CD, and led reliability reviews.
