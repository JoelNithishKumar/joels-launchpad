🛡️ ProgramOps Co-Pilot

AI-Driven Compliance & Risk Tracking for Engineering Operations

📌 Project Summary

Designed and implemented an AI-driven ProgramOps Co-Pilot to automate SOC2/GDPR/PII compliance checks across Slack, Jira, and Gmail — improving compliance readiness by 35% and reducing MTTR from 3 days to 2 days.

⚠️ Problem

Engineering teams struggled with fragmented compliance workflows: risk checks were manual, evidence was scattered across tools, and audits created recurring delays.

🎯 Objectives

  • Automate compliance artifact collection and risk flagging
  • Map dependencies between incidents, services, and controls
  • Reduce MTTR by surfacing risks early
  • Generate audit-ready evidence bundles automatically

⚙️ Solution Architecture

  • Orchestrated 6 specialized agents: Compliance, Dependency, Evaluation, Governance, Ingestion, Reporting
  • FastAPI microservices for modular pipelines
  • Pinecone semantic search to identify SOC2/GDPR/PII references
  • Slack, Jira, and Gmail API connectors for ingestion
  • Evidence bundles exported to GCS and surfaced in dashboards

🧩 Tech Stack

  • Backend: Python, FastAPI, Uvicorn
  • AI: LangChain, OpenAI GPT API, Pinecone (semantic search)
  • APIs: Slack API, Jira API, Gmail API
  • Infra: Docker, Google Cloud Storage
  • Monitoring: Prometheus + Grafana dashboards for MTTR/SLO metrics

⚡ Key Features

  • Real-time ingestion of compliance-related messages and tickets
  • Risk tagging using semantic + regex pattern matching
  • Dependency graph of controls and incidents for root cause tracing
  • Evidence package generation and delivery to auditors

🛠️ Implementation Details

  • Each agent has its own FastAPI service and message queue
  • Semantic retrieval over 200+ internal policy docs
  • Automated reports as PDF + CSV through Make.com workflows
  • CI pipeline: pytest + GitHub Actions + Docker build/push

📈 Results & Impact

  • Compliance readiness up by 35%
  • MTTR reduced from 3 days → 2 days
  • Significantly reduced audit preparation effort and legal risk

📌 My Role

Product & Tech Lead — defined risk taxonomy, designed agent architecture, implemented pipelines, and demoed to security/legal teams.

🚀 Future Improvements

  • Integrate SOC2 Type II automated evidence checks
  • Add risk scoring models using incident severity and frequency
  • Build user-facing compliance health dashboards
